git、gradle 常用命令

发表于 2021-05-12

合并多个 commit

例子：有待合并的多个 commit：001，002，003，004，005

1、reset 到 001 >>>> git add >>> git commit —amend >>> push 代码；

打标签

git tag -a v1.4 -m ‘my version 1.4’

git push origin v1.4

新增关联远程分支

git remote add remoteBranchName 仓库地址

e.g: git remote add remoteName(名字随意取) https://github.com/songmingwen/News-Comic-Reader.git

同步指定远程分支代码到本地分支

git pull –rebase <远程主机名> <远程分支名>

e.g: git pull —rebase zhihu release-5.14

查看关联的所有远程仓库

git remote -v

查看本地仓库关联的远程仓库

git branch -v / git branch -vv

删除远程分支

$ git push remoteName –delete

$ git push remoteName :

删除本地分支

$ git branch -D

删除本地过时分支

$ git remote prune origin

筛选指定分支

$ git branch -r |grep

拉分支修改并提交到该分支

$ git checkout -b remoteName/

$ git add .

$ git commit -m “”

$ git push remoteName :

推代码

git push <远程主机名> <本地分支名>:<远程分支名>

将某分支上的 commit(38361a68138140827b31b72f8bbfd88b3705d77a) 放到当前分支

$ git cherry-pick

合并多个 commit ：https://www.jianshu.com/p/964de879904a

$ git rebase -i <需要操作的前一个 commit id>

将log打印到文件

切换到一个盘如D盘

adb logcat -v time > log.txt

快速寻找当前焦点下的 Activity

adb shell dumpsys activity | find “mFocusedActivity”

美团热修复，用到的打包命令

gradlew clean assembleRelease –stacktrace –no-daemon

gradle 命令

./gradlew clean && ./gradlew assembleZhihuRelease –profile

./gradlew :app:uploadArchives 升级依赖版本
